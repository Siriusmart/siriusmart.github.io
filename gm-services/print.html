Goodmorning Services API Documentation

Overview

Goodmorning Services contains the api stuff, manages various accounts, storage related things.

Api structure

Structured similar to a REST api, all paths can be found in /api/[service]/vX/[path...].

Some requires posts requests, some get requests, and sometimes multipart post if the client is trying to upload a file.

Requirements

To run this, use cargo r (--release):

  • Copy dotenv-example to .env, and fill in the blanks.
  • Have mongodb running as specified in .env.

If you are using this as a dependency (such as gmt-server):

  • Run goodmorning_services::init() at the beginning.

Bindings

  • Rust

Accounts

/api/accounts

Account operations such as create, delete, login.

v1

/api/accounts/v1

POST /api/accounts/v1/create

Create an account

Request

{
  "username": String,
  "email": String,
  "password": String
}

Response

Status code: 201

{
  "type": "created",
  "id": i64,
  "token": String
}

Possible errors

  • invalid username
  • username taken
  • email taken
  • external

POST /api/accounts/v1/delete

Delete an account

Request

{
  "token": String
}

Response

Status code: 200

{
  "type": "deleted"
}

Possible errors

  • invalid token
  • external

POST /api/accounts/v1/login

Get user ID and token from identifier and password.

Request

{
  "identifier": String,
  "identifier-type": "id"/"username"/"email",
  "password": String
}

Response

Status code: 200

{
  "type": "login",
  "id": i64,
  "token": String
}

Possible errors

  • no such user
  • password incorrect
  • external

Storage

/api/storage

User storage operations such as copy, upload, ls.

v1

/api/storage/v1

POST /api/storage/v1/copy or /api/storage/v1/copy-overwrite

Copy file to destination path, file can be owned by self or by others (only if the file is hidden or public).

Request

{
  "token": String,
  "from-userid": i64, (who to copy from, can be your own id)
  "from": String,
  "to": String
}

Response

Status code: 201

{
  "type": "copied"
}

Possible errors

  • invalid token
  • not verified
  • permission denied
  • path occupied (non overwrite only)
  • file not found
  • file too large
  • external

GET /api/storage/v1/diritems/[token]/[path]

List items in directory.

For example to access /dir/deeper in your storage, you GET /api/storage/v1/diritems/token-gibberish/dir/deeper, where /dir/deeper is a directory.

Response

Status code: 200

{
  "type": "dir content",
  "content": [
    {
      "visibility": {
        "inherited": bool,
        "visibility": "hidden"/"private"/"public"
      },
      "is_file": bool,
      "name": String,
      "last_modified": u64,
      "size": u64
    },
    ...
  ]
}

Possible errors

  • invalid token
  • not verified
  • permission denied
  • file not found
  • type mismatch
  • external

Tex

/api/tex

GoodMorningTex required actions.

Compile

/api/tex/compile

Compile related actions - compile files from a source format (such as latex/md) to a readable format (pdf or html).

Valid compilations:

FromTo
markdownhtml

v1

/api/tex/compile/v1

Simple

POST /api/tex/compile/v1/simple

Compiles a source file to a web friendly format. Server will returns result when compilation is done.

Request

{
  "token": String,
  "path": String, (omit the beginning `/tex` as it can only be used in the `/tex` directory)
  "from": FromFormat,
  "to": ToFormat
}

Reference this page for available formats.

Response

Status code: 201

{
  "type": "compiled",
  "id": u64, // job id
  "newpath": String,
  "message": String
}

Possible errors

  • invalid token
  • not created
  • not verified
  • permission denied
  • external
  • compile error
